Реализация промиса

Особенности промиса

  1. Промис может менять состояние только один раз - то есть результатом может стать либо resolve, либо reject, выполненный один раз.
  2. Функция-executor выполняется синхронно. Если ф-ция-executor не содержит асинхронных операций, то resolve/reject также вызовется синхронно.
  3. Функция resolve/reject чаще всего вызывается асинхронно(setTimeout, fetch), поэтому получить результат выполнения промиса можно тоже только асинхронно.
  4. Метод then принимает на вход 2 коллбэка:
    1. На случай, если промис завершен успешно (отработала ф-ция resolve).
    2. На случай, если промис завершен с ошибкой (отработала ф-ция reject).
    Эти коллбэки чаще всего принимают в качестве параметра результат работы функций resolve/catch.
    Однако, зачастую в then передают только первый коллбэк, оставляя обработку ошибок блоку catch.
  5. Метод catch предназначен для перехвата ошибок. Все ошибки, которые произойдут в промисе и последующей цепочке промисов then, будут отловлены методом catch.
  6. Промис, используемый повторно, отдаст коллбэкам одно и то же значение.
